# pembelajaran-1-materi
A
Algoritma: Sekumpulan instruksi atau aturan yang terdefinisi dengan baik untuk memecahkan masalah atau melakukan perhitungan.
Anomali: Nilai atau pola data yang menyimpang secara signifikan dari mayoritas data lainnya, sering dideteksi selama tahap Exploratory Data Analysis (EDA).
API: Antarmuka pemrograman aplikasi adalah seperangkat aturan yang memungkinkan dua perangkat lunak berbeda untuk berkomunikasi.
Artificial Intelligence (AI): Bidang ilmu komputer yang bertujuan menciptakan sistem yang mampu meniru kecerdasan manusia untuk melakukan tugas tertentu.
Automatic Code Completions: Fitur pada editor kode yang menyarankan dan menyelesaikan baris kode secara otomatis saat pengguna mengetik.
 

B
Bahasa Alami: Bahasa yang digunakan manusia dalam komunikasi sehari-hari, yang dapat diproses dan dipahami oleh komputer.
Batch Scoring: Proses menghasilkan prediksi untuk sejumlah besar data secara bersamaan, biasanya pada interval waktu yang ditetapkan.
Bias: Ketidakseimbangan atau kesalahan sistematis dalam model atau data yang menyebabkan hasil prediksi menyimpang dari kondisi sebenarnya.
Business Intelligence: Proses dan teknologi untuk mengumpulkan, menyimpan, dan menganalisis data bisnis untuk membuat keputusan yang lebih baik.
 

C
Cell: Unit dasar dalam sebuah notebook yang dapat berisi narasi (markdown) atau kode yang dapat dieksekusi.
Churn: Istilah bisnis yang mengacu pada kecenderungan pelanggan atau pengguna untuk berhenti menggunakan layanan atau produk, sering dijadikan variabel target dalam predictive analysis.
Classification: Jenis model machine learning yang memprediksi nilai kategorikal, seperti menentukan apakah suatu objek termasuk dalam kelas tertentu.
Clustering: Jenis model machine learning yang mengelompokkan titik data yang memiliki kemiripan ke dalam grup tanpa adanya label kategori di awal.
Cloud Data Sources: Sumber data yang disimpan di infrastruktur komputasi awan, seperti layanan penyimpanan data online.
Clusters: Kelompok titik data yang terbentuk dalam proses clustering yang memiliki karakteristik atau kemiripan yang serupa.
Code-first: Pendekatan pengembangan yang memprioritaskan penulisan kode secara manual untuk membangun suatu solusi.
Code Snippet: Bagian kecil dari kode sumber yang dapat digunakan kembali untuk tugas pemrograman umum atau spesifik.
Compute Engines: Komponen perangkat lunak yang bertugas melakukan perhitungan dan pemrosesan data dalam suatu platform analitik.
Compute Resources: Sumber daya komputasi, seperti CPU dan memori, yang digunakan untuk menjalankan tugas pemrosesan data atau model.
CSV: Format file teks sederhana yang menyimpan data tabular di mana setiap nilai dipisahkan oleh koma.
 

D
DataFrame: Struktur data berbentuk tabel dua dimensi (baris dan kolom) yang digunakan untuk memuat, memanipulasi, dan menganalisis data di lingkungan notebook Fabric.
Data Distribution: Sebaran nilai-nilai dalam sebuah himpunan data yang menunjukkan seberapa sering setiap nilai muncul.
Data Engineering: Disiplin ilmu yang berfokus pada perancangan dan pembangunan sistem untuk mengumpulkan, menyimpan, dan mentransformasi data mentah.
Data Factory: Layanan dalam Fabric yang digunakan untuk mengambil, mentransformasi, dan mengorkestrasi pergerakan dan pemrosesan data.
Data Ingestion: Proses memuat atau mentransfer data dari satu atau lebih sumber ke sistem penyimpanan.
Data Integration: Proses menggabungkan data dari sumber yang berbeda menjadi satu tampilan yang seragam dan kohesif.
Data Mesh: Paradigma arsitektur data terdistribusi yang memperlakukan data sebagai produk, di mana kepemilikan dan penyediaan data dipegang oleh domain bisnis.
Data Science: Bidang interdisipliner yang menggunakan metode ilmiah, proses, algoritma, dan sistem untuk mengekstrak pengetahuan dan wawasan dari data.
Data Visualization: Representasi data secara grafis, seperti melalui bagan atau plot, untuk mempermudah pemahaman pola dan tren.
Data Warehousing: Sistem yang mengumpulkan, menyimpan, dan mengelola data historis dari berbagai sumber untuk mendukung analisis bisnis.
Data Wrangler: Alat bantu visual interaktif yang terintegrasi di notebook Microsoft Fabric untuk eksplorasi, pembersihan, dan transformasi data.
Delta Table: Tabel yang menggunakan format Delta Lake untuk penyimpanan data, menawarkan kemampuan transaksi ACID, versioning, dan skema yang fleksibel, dan merupakan dasar dari OneLake dan Lakehouse di Fabric.
delta-parquet: Format penyimpanan data default di OneLake untuk data tabular, yang menggabungkan format Parquet dengan kapabilitas transaksi Delta Lake.
Dependency: Modul, library, atau paket perangkat lunak lain yang dibutuhkan oleh sebuah program atau proyek agar dapat berfungsi dengan baik.
Deployment: Proses peletakan model machine learning yang telah dilatih ke dalam lingkungan produksi agar dapat digunakan untuk menghasilkan prediksi.
 

E
End-to-End: Menggambarkan proses atau sistem yang lengkap dan terintegrasi, mencakup setiap tahapan dari awal hingga akhir.
Error Marking: Fitur dalam editor kode yang secara visual menyoroti kesalahan sintaksis atau logis dalam kode yang sedang ditulis.
ETL (Extract, Transform, Load): Proses integrasi data yang meliputi pengambilan data dari sumber, perubahan format dan strukturnya, lalu pemuatan ke sistem tujuan.
Experiments: Konsep dalam machine learning yang melacak dan merekam berbagai upaya pelatihan model dengan pengaturan yang berbeda.
Exploratory Data Analysis (EDA): Langkah awal dalam proyek data science untuk memahami karakteristik utama, pola, dan anomali dalam data melalui visualisasi dan statistik.
 

F
Feature Engineering: Proses menciptakan fitur input baru yang informatif dari data mentah yang sudah ada untuk meningkatkan akurasi model machine learning.
Feature Selection: Proses memilih subset variabel input (fitur) yang paling relevan untuk digunakan dalam pelatihan model machine learning.
Fitur: Variabel input atau atribut yang digunakan oleh model machine learning untuk membuat prediksi.
Forecasting: Jenis model machine learning yang memprediksi nilai numerik kontinu di masa depan, biasanya didasarkan pada data deret waktu.
Fragmentasi: Keadaan di mana data tersebar di berbagai sistem atau platform yang berbeda, yang menyebabkan tantangan dalam analitik yang terpadu.
Framework: Kerangka kerja perangkat lunak yang menyediakan struktur dan fungsi dasar untuk mengembangkan aplikasi atau solusi tertentu.
 

G
Gateway: Komponen yang berfungsi sebagai jembatan aman untuk mentransfer data antara layanan cloud Fabric dan sumber data yang berada di lingkungan on-premise.
Generative AI: Sub-bidang Artificial Intelligence yang berfokus pada pembangunan sistem yang dapat menghasilkan konten baru, seperti teks, gambar, atau kode.
 

H
Histogram: Jenis bagan yang menampilkan distribusi frekuensi data numerik dengan membagi data menjadi "bin" atau interval.
Hyperparameters: Parameter yang nilainya ditetapkan secara manual sebelum proses pelatihan model machine learning dimulai.
 

I
Imputasi: Teknik yang digunakan untuk mengganti nilai yang hilang (missing data) dalam suatu dataset dengan nilai yang diestimasi.
Ingestion: Istilah lain untuk Data Ingestion, yaitu proses memuat data ke dalam sistem analitik.
Insight: Wawasan atau pemahaman berharga yang diperoleh dari analisis data.
Intelligent Code Completion: Fitur yang menyarankan penyelesaian kode secara kontekstual dan lebih cerdas, sering kali didukung oleh Artificial Intelligence.
 

J
JSON: Format file teks ringan dan mudah dibaca manusia yang umum digunakan untuk pertukaran data semi-terstruktur.
 

K
Komponen Utama: Variabel baru yang tidak berkorelasi, dihasilkan dari Principal Component Analysis (PCA), yang menangkap sebagian besar variasi dalam data asli.
Korelasi: Hubungan statistik antara dua variabel yang menunjukkan sejauh mana perubahan pada satu variabel berkaitan dengan perubahan pada variabel lainnya.
 

L
Lakehouse: Arsitektur data modern di Fabric yang menggabungkan kemampuan data warehouse dengan fleksibilitas data lake.
Library: Sekumpulan kode dan fungsi yang telah ditulis sebelumnya yang dapat digunakan oleh programmer untuk mempermudah pengembangan perangkat lunak.
Line Plot: Jenis bagan yang menampilkan titik-titik data yang dihubungkan dengan segmen garis lurus, sangat efektif untuk data deret waktu.
Logical Lake: Lapisan konseptual di atas penyimpanan data fisik yang menyatukan data dari berbagai sumber tanpa duplikasi fisik.
Low-Code: Pendekatan pengembangan perangkat lunak yang memungkinkan pengguna membuat aplikasi atau alur kerja dengan sedikit atau tanpa penulisan kode.
 

M
Machine Learning: Sub-bidang Artificial Intelligence yang memungkinkan sistem belajar dari data, mengidentifikasi pola, dan membuat keputusan tanpa diprogram secara eksplisit.
Metadata: Data tentang data yang menjelaskan struktur, skema, asal, dan karakteristik lain dari suatu item data dalam Fabric.
Metrics: Nilai kuantitatif yang digunakan untuk mengukur performa, akurasi, dan kualitas suatu model machine learning selama atau setelah pelatihan, seperti R-squared atau accuracy.
MLflow: Framework open-source terintegrasi di Microsoft Fabric yang digunakan untuk mengelola siklus hidup machine learning, termasuk pelacakan experiments dan metrik, serta penyimpanan model artifacts.
Missing Completely at Random (MCAR): Situasi di mana data yang hilang tidak memiliki hubungan dengan nilai variabel apa pun, baik yang hilang maupun yang diamati.
Missing at Random (MAR): Situasi di mana data yang hilang memiliki hubungan dengan variabel lain yang diamati, tetapi tidak dengan nilai yang hilang itu sendiri.
Missing Data: Fenomena ketika beberapa nilai untuk variabel tertentu tidak tersedia dalam sebuah dataset.
Missing Not at Random (MNAR): Situasi di mana data yang hilang berhubungan langsung dengan nilai dari data yang hilang itu sendiri.
Model: Representasi matematis yang menangkap pola dan hubungan yang telah dipelajari dari data untuk membuat prediksi atau keputusan.
Model Artifacts: Semua file pendukung yang dihasilkan selama pelatihan model, seperti berkas bobot model itu sendiri dan berkas MLmodel yang mendefinisikan cara kerja model, disimpan dan dilacak oleh MLflow.
Modeling/Pemodelan: Tahap dalam alur kerja data science di mana algoritma dipilih dan dilatih menggunakan data yang telah disiapkan untuk menemukan pola dan membuat prediksi.
Model Signature: Skema atau kontrak yang mendefinisikan format masukan (input) dan keluaran (output) yang diharapkan dari model machine learning, dicatat dalam berkas MLmodel.
Multivariat: Analisis atau data yang melibatkan dan menguji hubungan antara lebih dari dua variabel secara bersamaan.
 

N
Normalisasi: Teknik pra-pemrosesan data untuk menskalakan nilai-nilai numerik dalam dataset ke rentang standar, biasanya antara 0 dan 1.
Notebook: Lingkungan interaktif berbasis web di Fabric yang memungkinkan pengguna menulis dan menjalankan kode bersama dengan narasi (markdown) dalam cell.
 

O
One-hot encoding: Teknik preprocessing yang mengubah variabel kategorikal menjadi format numerik biner yang dapat diproses oleh algoritma machine learning.
ONNX (Open Neural Network Exchange): Format standar terbuka yang memungkinkan model machine learning ditransfer dan digunakan di antara framework yang berbeda, memastikan konsistensi deployment.
On-premise: Model penyebaran perangkat lunak di mana sistem diinstal dan dioperasikan dari lokasi fisik organisasi sendiri.
Open-source: Perangkat lunak yang kode sumbernya dapat dilihat, dimodifikasi, dan didistribusikan secara bebas oleh siapa pun.
P
Pair Plot: Matriks plot yang menampilkan hubungan antara setiap pasangan variabel numerik dalam dataset dan distribusi masing-masing variabel pada diagonalnya.
Pemrosesan Paralel: Kemampuan suatu sistem komputasi untuk menjalankan banyak instruksi secara bersamaan di beberapa unit pemrosesan.
Persentil: Ukuran statistik yang menunjukkan nilai di bawahnya persentase tertentu dari observasi dalam suatu kelompok berada.
Pipelines: Rangkaian langkah yang terotomasi dan terstruktur untuk memindahkan dan memproses data dari sumber hingga tujuan.
Power BI: Layanan dalam Fabric yang dikhususkan untuk visualisasi data, pembuatan laporan interaktif, dan dashboard untuk analisis bisnis.
Pre-processing: Tahap awal dalam data science yang melibatkan pembersihan, transformasi, dan persiapan data mentah agar siap dan optimal untuk dianalisis atau digunakan untuk melatih model machine learning.
Principal Component Analysis (PCA): Teknik statistik untuk mereduksi dimensi data dengan mengubah sekumpulan variabel yang berkorelasi menjadi sekumpulan komponen utama yang tidak berkorelasi.
Processing: Serangkaian operasi yang dilakukan pada data untuk mengubah atau mengekstrak informasi yang bermakna.
Predictive Analysis: Cabang analitik data yang menggunakan data historis untuk memprediksi hasil atau tren di masa depan.
 

R
R-squared (R2): Metrik evaluasi yang digunakan dalam regression untuk menunjukkan proporsi varians variabel dependen yang dapat diprediksi dari variabel independen.
Regression: Jenis model machine learning yang memprediksi nilai numerik kontinu, seperti harga atau suhu.
Real-time Intelligence: Kemampuan untuk mengumpulkan, menganalisis, dan mengambil tindakan berdasarkan data yang masuk secara instan.
Run: Sebuah instans tunggal dari eksekusi tugas atau pelatihan model dalam sebuah experiment machine learning.
 

S
Scalable: Kemampuan suatu sistem untuk menangani peningkatan beban kerja atau volume data tanpa penurunan performa.
Scatter Plot: Jenis bagan yang menggunakan titik-titik untuk menampilkan nilai dari dua variabel, berguna untuk mengidentifikasi korelasi.
scikit-learn: Library open-source Python yang populer dan terintegrasi di Fabric, digunakan untuk melatih model machine learning tradisional.
Sensitivity Label: Label yang diterapkan pada item data Fabric untuk mengklasifikasikan tingkat kerahasiaan dan menerapkan kebijakan perlindungan data.
Silo: Kondisi di mana data terisolasi dan tidak dapat diakses atau dibagikan secara bebas antara berbagai departemen atau sistem, yang menjadi hambatan analitik.
Skewed: Distribusi data yang tidak simetris, di mana sebagian besar data berkumpul di satu sisi.
Software-as-a-Service (SaaS): Model penyediaan perangkat lunak di mana aplikasi di-hosting oleh penyedia dan diakses pengguna melalui internet.
Spark Compute: Sumber daya komputasi yang didukung oleh framework Apache Spark, dirancang untuk pemrosesan data skala besar.
Standardisasi: Teknik pra-pemrosesan data yang menskalakan nilai-nilai sehingga memiliki rata-rata nol dan deviasi standar satu.
Status Data Refresh: Indikator yang menunjukkan keberhasilan atau kegagalan pembaruan data terbaru pada dataset atau item analitik, serta waktu pembaruan tersebut.
Syntax Highlighting: Fitur editor kode yang menampilkan teks dalam berbagai warna dan font berdasarkan kategori istilah, yang meningkatkan keterbacaan kode.
 

T
Tabular: Format data yang terstruktur dalam bentuk tabel baris dan kolom.
Time-series Data: Kumpulan observasi data yang diurutkan atau diindeks berdasarkan waktu.
Transform: Proses mengubah struktur, format, atau nilai data agar sesuai untuk analisis atau pelatihan model, bagian dari proses preprocessing.
Training: Proses melatih model machine learning dengan memberinya data input untuk belajar dari pola yang ada.
 

U
Univariat: Analisis statistik yang hanya melibatkan dan berfokus pada satu variabel pada satu waktu.
Upstream: Mengacu pada tahapan atau sistem sebelumnya dalam alur kerja pemrosesan data, tempat data berasal.
V
Variabel Target: Variabel yang ingin diprediksi atau dijelaskan oleh model machine learning berdasarkan fitur input.
Version Control: Sistem yang mencatat perubahan pada serangkaian file seiring waktu sehingga Anda dapat meninjau kembali versi tertentu.
Variables Explorer: Fitur dalam notebook yang memungkinkan pengguna untuk melacak dan melihat nilai, tipe, dan properti semua variabel yang sedang aktif secara real-time.
W
Workload: Jumlah pekerjaan atau tugas pemrosesan yang harus dilakukan oleh suatu sistem komputasi dalam periode waktu tertentu.
Workspace: Wadah logis dalam platform yang berfungsi sebagai ruang kerja atau studio proyek untuk mengatur dan mengelola aset data dan analitik.
